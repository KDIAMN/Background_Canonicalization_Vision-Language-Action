Open-loop 실행 구조 때문에 모델은 사실 8 스텝마다 한 번만 이미지를 봅니다:


T+0: action_queue 비었음 → 모델 호출 (마스킹된 프레임 입력) → 액션 8개 생성
T+1~7: action_queue에서 꺼내서 실행 → 모델 호출 안 함 (이미지 안 봄)
T+8: action_queue 비었음 → 모델 호출 (마스킹된 프레임 입력) → ...
즉 T+1~7 프레임은 마스킹을 해도 안 해도 모델 결정에 영향을 주지 않습니다. 영상 녹화용으로만 쓰입니다.

학습 분포와 일치시키려면 모델이 보는 프레임 (매 8번째)만 마스킹하면 충분하고, 그게 8배 빠릅니다.
